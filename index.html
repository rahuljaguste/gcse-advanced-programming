Chapter 1: Subroutines

Procedures & Functions

"Don't repeat yourself. Package reusable instructions into named blocks."

Imagine you're teaching someone to make pizza. You wouldn't explain "how to make dough" every single time. You'd say "Step 1: Make the dough" and they'd follow that recipe.

A procedure is like a recipe step — "do this thing" (no answer expected back).

A function is like asking a question that gives back an answer — "How many slices do we need for 4 people?" → returns 8.

Key Concepts

TermWhat it isPizza Example
ProcedureBlock of code that does something (no return value)make_dough() — just makes it
FunctionBlock of code that calculates and returns a valueslices_needed(people) → returns a number
ParameterInput the subroutine needs to workpeople in slices_needed(people)
Return valueThe answer a function gives backThe number 8
CallingRunning the subroutine by its namemake_dough()

Live Demo — Type This!

python — pizza_shop.py
# PROCEDURE — does something, returns nothing
def greet(name):
    print(f"Hey {name}! Welcome to the pizza shop!")

# FUNCTION — calculates and returns a value
def slices_needed(people):
    return people * 2  # 2 slices per person

# USING THEM
greet("Alex")
total = slices_needed(4)
print(f"We need {total} slices!")

# A function can use another function
def pizzas_needed(people):
    slices = slices_needed(people)
    return slices // 8  # 8 slices per pizza

pizzas = pizzas_needed(5)
print(f"Order {pizzas} pizzas!")
# Think about it: why does this say 1 not 2?
# What about the remaining 2 slices?

Pseudocode (Important for Exams!)

pseudocode
SUBROUTINE greet(name)
    OUTPUT "Hey " + name + "! Welcome to the pizza shop!"
END SUBROUTINE

FUNCTION slices_needed(people)
    RETURN people * 2
END FUNCTION

greet("Alex")
total ← slices_needed(4)
OUTPUT "We need " + total + " slices!"

Chapter 2: 1D Arrays

Data Structures & Lists

"Store a collection of related items under ONE name instead of making dozens of variables."

Your phone's contact list is an array. Instead of writing each friend's name on a separate piece of paper (separate variables), you have ONE list called "Contacts" and each person has a position number.

The first contact is at position 0 (not 1!), the second at position 1, and so on.

Key Concepts

ConceptMeaningExample
Array/ListOrdered collection under one namefriends = ["Aarav", "Zain", "Lily"]
IndexPosition number (starts at 0!)friends[0] → "Aarav"
LengthHow many itemslen(friends) → 3
AppendAdd to the endfriends.append("Mia")
RemoveDelete a specific itemfriends.remove("Zain")
SlicingCut out a portionfriends[0:2] → first two items

Live Demo — Type This!

python — arrays_demo.py
# Imagine these are scores from a game
scores = [45, 78, 92, 33, 61]

# Find the highest score (THE MANUAL WAY — understand this for exams!)
max_score = 0
for i in range(len(scores)):
    if scores[i] > max_score:
        max_score = scores[i]
print(f"Highest score: {max_score}")

# Now the Python shortcut
print(f"Highest (shortcut): {max(scores)}")

# Slicing — get top 3 after sorting
scores.sort(reverse=True)
top_3 = scores[0:3]
print(f"Top 3 scores: {top_3}")

# Looping and displaying nicely
friends = ["Aarav", "Zain", "Lily", "Mia"]
for i in range(len(friends)):
    print(f"Position {i}: {friends[i]}")

Pseudocode

pseudocode
scores ← [45, 78, 92, 33, 61]

max_score ← 0
FOR index ← 0 TO LEN(scores) - 1
    IF scores[index] > max_score THEN
        max_score ← scores[index]
    END IF
END FOR
OUTPUT "Highest score: " + max_score

Chapter 3: 2D Arrays

Tables, Grids, and Lists of Lists

"A list of lists — like a spreadsheet or a table with rows and columns."

Think of a cinema seating chart. Row 0 has seats [A, B, C, D]. Row 1 has seats [A, B, C, D]. To find a specific seat, you need TWO numbers: which row, and which seat in that row.

That's exactly how a 2D array works: cinema[row][seat]

Number of rowslen(seats)
Number of columnslen(seats[0])

Live Demo — Type This!

python — noughts_and_crosses.py
# Noughts and Crosses board — a 2D array!
board = [
    [" ", " ", " "],
    [" ", " ", " "],
    [" ", " ", " "]
]

def print_board(board):
    print("  0   1   2")
    for i, row in enumerate(board):
        print(f"{i} " + " | ".join(row))
        if i < 2:
            print("  ---------")

# Place some moves
board[1][1] = "X"  # Centre
board[0][0] = "O"  # Top-left
board[2][2] = "X"  # Bottom-right

print_board(board)
# Try changing the positions and see what happens!

Chapter 4: Records

Grouping Different Types of Data Together

"Group DIFFERENT types of data about ONE thing together."

Think of your school ID card. It has your name (text), your year group (number), your photo (image), and your student ID (code). These are all DIFFERENT types of data, but they all describe ONE person.

An array says "here are 10 names." A record says "here's everything about ONE student."

Key Concepts

ConceptArrayRecord
StoresMany items of the SAME typeMultiple items of DIFFERENT types
Access byIndex number [0]Field name ["name"]
Example["Aarav", "Zain"]{"name": "Aarav", "age": 14}

Live Demo — Type This!

python — records_demo.py
# A record using a dictionary
student = {
    "name": "Alex",
    "age": 14,
    "year_group": 10,
    "favourite_subject": "Computing"
}

# Accessing fields
print(f"Name: {student['name']}")
print(f"Age: {student['age']}")

# An ARRAY of RECORDS — super powerful!
students = [
    {"name": "Alex", "age": 14, "score": 92},
    {"name": "Aarav", "age": 15, "score": 87},
    {"name": "Lily", "age": 14, "score": 95},
]

# Find the student with the highest score
best = students[0]
for student in students:
    if student["score"] > best["score"]:
        best = student
print(f"Top student: {best['name']} with {best['score']}!")

Pseudocode

pseudocode
RECORD Student
    name: STRING
    age: INTEGER
    score: INTEGER
END RECORD

students ← ARRAY OF Student
best ← students[0]
FOR index ← 0 TO LEN(students) - 1
    IF students[index].score > best.score THEN
        best ← students[index]
    END IF
END FOR
OUTPUT "Top student: " + best.name

Chapter 5: External Files

Reading and Writing Data to Files

"Save data so it survives after the program closes. Read data that already exists."

Your brain is like RAM — it forgets things when you sleep (program closes). A notebook is like a file — you write things down so you can read them tomorrow.

Programs work the same way: variables disappear when the program ends, but files stick around forever.

Key Concepts

OperationPython CodeWhat it does
Open for writingf = open("data.txt", "w")Creates/overwrites file
Open for appendingf = open("data.txt", "a")Adds to end (safe!)
Open for readingf = open("data.txt", "r")Read the contents
Write a linef.write("Hello\n")Puts text in file
Read all lineslines = f.readlines()Gets all lines as a list
Closef.close()Always close when done!

Critical rule: "w" ERASES the file first. "a" ADDS to the end. Mix these up and you lose data! When in doubt, use "a".

Live Demo — Type This!

python — file_demo.py
# WRITE a high score to a file
name = input("Enter your name: ")
score = input("Enter your score: ")

f = open("highscores.txt", "a")  # 'a' = append, won't erase old scores!
f.write(f"{name} {score}\n")
f.close()
print("Score saved!")

# READ all high scores
f = open("highscores.txt", "r")
lines = f.readlines()
f.close()

print("\n--- HIGH SCORES ---")
for line in lines:
    print(line.strip())  # .strip() removes the \n at the end

Pseudocode

pseudocode
myFile ← OPEN_FILE("highscores.txt")
lines ← READ_LINES(myFile)
FOR EACH line IN lines
    OUTPUT line
END FOR
CLOSE_FILE(myFile)

Chapter 6: Random Numbers

Games, Simulations, and Unpredictability

"Let the computer make unpredictable choices — essential for games, simulations, and security."

When you roll a dice, you can't predict the number. random.randint(1,6) is a digital dice.

When you shuffle a deck of cards, you use random.shuffle(). When you pick a random card, you use random.choice().

Games without randomness would be boring — you'd know exactly what happens every time!

Key Concepts

python — random_cheatsheet.py
import random

my_list = [1, 2, 3, 4, 5]

print(random.randint(1, 6))        # Random integer between 1 and 6 (inclusive)
print(random.choice(["a", "b"]))   # Pick one random item from a list
random.shuffle(my_list)             # Shuffle a list in-place
print("Shuffled:", my_list)
print(random.random())              # Random decimal between 0.0 and 1.0
print(random.sample(my_list, 3))    # Pick 3 unique random items from a list

Live Demo — Type This!

python — dice_demo.py
import random

# Digital dice
for i in range(5):
    roll = random.randint(1, 6)
    print(f"Roll {i+1}: {roll}")

# Random name picker (for who does the washing up!)
family = ["Mum", "Dad", "Alex", "Sister"]
chosen = random.choice(family)
print(f"\n{chosen} is doing the washing up tonight!")

# Coin flip simulator
heads = 0
tails = 0
for _ in range(100):
    if random.randint(0, 1) == 0:
        heads += 1
    else:
        tails += 1
print(f"\n100 coin flips: {heads} heads, {tails} tails")

Chapter 7: Scope & Structured Programming

Where Variables Live and How to Organise Code

"Where you create a variable determines where you can use it. Structure your code into logical blocks."

Think of your house. Your bedroom is your "local scope" — your stuff stays there. The kitchen is "shared scope" — everyone uses it.

A local variable is like your phone charger in your room — only you can use it. A global variable is like the WiFi password — everyone in the house knows it.

If a friend comes over (a function is called), they can't access your bedroom stuff unless you hand it to them (pass it as a parameter).

Key Concepts

TermMeaningExample
Local variableCreated inside a function, only usable theredef greet(): name = "Alex"
Global variableCreated outside all functions, usable everywherescore = 0 at the top
global keywordLets a function modify a global variableglobal score
Structured programmingUsing sequence, selection, iterationClean, logical code flow

The Three Structures

StructureWhat it isPython Example
SequenceCode runs line by line, in orderx = 1 then y = 2 then print(x + y)
SelectionMaking decisions (branching)if age >= 18:
IterationRepeating code (loops)for i in range(10): or while True:

Live Demo — Type This!

python — scope_demo.py
score = 0  # GLOBAL variable — lives outside all functions

def add_points(points):
    global score           # Without this line, Python would create a LOCAL 'score'
    score = score + points
    bonus = points * 2     # LOCAL variable — only exists inside this function
    print(f"Bonus calculated: {bonus}")

add_points(10)
print(f"Score is now: {score}")    # Works! score is global

# Uncomment the next line to see an error:
# print(bonus)  # ERROR! bonus is local to add_points

# WITHOUT global keyword — common mistake!
lives = 3

def lose_life():
    # lives = lives - 1  # This would ERROR because Python thinks
    #                     # you're trying to create a LOCAL 'lives'
    global lives
    lives = lives - 1

lose_life()
print(f"Lives remaining: {lives}")  # 2

Chapter 8: Validation, Authentication & Testing

Making Programs Robust and Secure

"Never trust user input. Always check it. Protect your system with passwords. Test everything."

Think of a nightclub bouncer.

Validation is the bouncer checking your age — "Are you 18 or over?" If you say 300, the range check catches that.

Authentication is the bouncer checking your name is on the guest list.

Testing is the club owner pretending to be underage to make sure the bouncer actually does their job!

Three Types of Validation

CheckWhat it doesExample
PresenceIs the field empty?Username can't be blank
RangeIs it within limits?Age must be 0-120
LengthRight number of characters?Password must be 8+ chars

Three Types of Tests

Test TypePurposeExample for age >= 18
NormalExpected inputAge = 25 (should be accepted)
BoundaryEdge valuesAge = 18 (right on the limit)
ErroneousWrong/impossible inputAge = -5, or "hello"

GIGO = Garbage In, Garbage Out. If you put rubbish data in, you get rubbish results out. Validation stops the garbage getting in!

Live Demo — Type This!

python — validation_demo.py
# A ROBUST age checker with all three validation types
def get_valid_age():
    while True:
        age_input = input("Enter your age: ")

        # PRESENCE CHECK — is it empty?
        if age_input == "":
            print("You must enter something!")
            continue

        # TYPE CHECK — is it a number?
        if not age_input.isdigit():
            print("That's not a number!")
            continue

        age = int(age_input)

        # RANGE CHECK — is it sensible?
        if age < 0 or age > 120:
            print("Age must be between 0 and 120!")
            continue

        return age  # Only reaches here if ALL checks pass

age = get_valid_age()
print(f"Your age is {age}. Valid!")

# AUTHENTICATION — simple login
correct_user = "Alex"
correct_pass = "Python123"

username = input("\nUsername: ")
password = input("Password: ")

if username == correct_user and password == correct_pass:
    print("Access granted! Welcome!")
else:
    print("Access denied!")

Chapter 9: Programming Languages

How Code Actually Runs on Hardware (Theory)

"Computers only understand 1s and 0s. Everything you write in Python gets translated."

Imagine you speak English and need to instruct someone who only speaks binary.

Machine code = speaking binary directly (exhausting).
Assembly language = using abbreviations like texting: "brb", "lol" (slightly easier).
Python = speaking normal English (easy for you, but needs a translator).

The compiler translates the whole book before the person reads it.
The interpreter translates one sentence at a time, waiting for them to finish each one.

Language Levels

LevelLanguageHuman-Friendly?Speed
Low LevelMachine Code (0s and 1s)Not at allVery fast
Low LevelAssembly (ADD, MOV, SUB)A bitFast
High LevelPython, Java, C++Yes!Needs translating first

Translators

TranslatorWhat it doesKey fact
AssemblerAssembly → Machine CodeOne-to-one translation (each instruction becomes one machine code instruction)
CompilerTranslates ALL high-level code at onceCreates a standalone executable file. Used for distributing finished software.
InterpreterTranslates line by line, runs immediatelySlower but great for testing/debugging. Python uses this!

Machine Code Structure

Each machine code instruction has two parts:

  • OPCODE — the operation to perform (e.g., ADD, LOAD, STORE)
  • OPERAND — the data or memory address to use

Example: 1010 0011 → OPCODE 1010 (ADD) + OPERAND 0011 (memory location 3)

Compiler vs Interpreter

FeatureCompilerInterpreter
TranslatesWhole program at onceOne line at a time
SpeedFaster to run (already translated)Slower (translates every time)
ErrorsShows all errors after compilingStops at the first error
DebuggingHarder (must recompile)Easier (change and re-run)
OutputCreates an executable fileNo executable file
ExamplesC, C++, JavaPython, JavaScript

Fun Activity: Human Computer Game

Do this with your uncle — no computer needed!

  1. Round 1 (Machine Code): One person gives binary commands: 1 = stand, 0 = sit. Shout "1, 0, 1, 1, 0" — the other must stand/sit.
  2. Round 2 (Assembly): Use mnemonics: "STN" = stand, "SIT" = sit. A third person (the Assembler) translates to binary.
  3. Round 3 (High-Level): Use English: "Do the morning exercise." The Compiler converts the whole thing at once.

Discussion: Which was easiest for the programmer? Which was fastest for the CPU? That's the trade-off!

Chapter 12: Algorithm Efficiency

Unit 2 — Comparing algorithms and understanding efficiency

"There's often more than one way to solve a problem — but some ways are faster than others."

Imagine you need to find a word in a dictionary. You could start at page 1 and check every page (slow). Or you could open it roughly in the middle and narrow down from there (fast). Both work, but one is massively more efficient.

Key Concepts

ConceptMeaning
EfficiencyHow quickly an algorithm solves a problem, or how few resources (time, memory) it uses
Time complexityHow the number of steps grows as the input gets bigger
ComparisonsThe number of times the algorithm checks or compares values — fewer = more efficient

Comparing Algorithms

Two algorithms that solve the same problem can have very different efficiency. We compare them by counting how many steps (comparisons, swaps, etc.) they need for the same input. For example:

  • Linear search on 1,000 items: up to 1,000 comparisons (worst case)
  • Binary search on 1,000 items: up to 10 comparisons (worst case)

Same problem, hugely different efficiency!

Live Demo — Two Ways to Find the Max

python — efficiency_demo.py
# Method 1: Manual loop (we control every step)
numbers = [45, 78, 12, 92, 33, 67, 85, 21, 56, 90]
max_val = numbers[0]
comparisons = 0
for num in numbers:
    comparisons += 1
    if num > max_val:
        max_val = num
print(f"Max: {max_val} (took {comparisons} comparisons)")

# Method 2: Sort first, then take last (works but wasteful!)
import time
numbers2 = [45, 78, 12, 92, 33, 67, 85, 21, 56, 90]
numbers2.sort()  # sorting does MANY comparisons internally
max_val2 = numbers2[-1]
print(f"Max: {max_val2} (sorted entire list just to find max!)")
print("\nMethod 1 is more efficient — it only needs N comparisons.")

Chapter 13: Searching Algorithms

Unit 2 — Linear search and binary search

"Finding something in a list — the brute-force way vs the smart way."

Linear search is like looking for a book on an unsorted shelf — you check every single book from left to right until you find it.

Binary search is like looking up a word in a dictionary — you open the middle, decide if the word is before or after, and cut the remaining pages in half each time. But the dictionary MUST be in alphabetical order (sorted)!

def linear_search(data, target):
    for i in range(len(data)):
        if data[i] == target:
            return i  # Found! Return the index
    return -1  # Not found

python — binary_search.py
def binary_search(data, target):
    low = 0
    high = len(data) - 1
    steps = 0

    while low <= high:
        mid = (low + high) // 2
        steps += 1
        print(f"  Step {steps}: checking index {mid} = {data[mid]}")

        if data[mid] == target:
            return mid, steps
        elif data[mid] < target:
            low = mid + 1    # target is in the RIGHT half
        else:
            high = mid - 1   # target is in the LEFT half

    return -1, steps

numbers = [3, 7, 12, 19, 25, 31, 42, 56, 78, 92]
print("Searching for 42:")
index, steps = binary_search(numbers, 42)
print(f"Found at index {index} in {steps} steps!")

print("\nSearching for 99:")
index, steps = binary_search(numbers, 99)
print(f"Result: {index} ({steps} steps — not found)")

Chapter 14: Sorting Algorithms

Unit 2 — Bubble sort and merge sort

"Putting things in order — the simple way and the clever way."

Bubble sort is like sorting a line of students by height. You compare neighbours, swap if they're in the wrong order, and repeat until nobody needs swapping. The tallest person "bubbles" to the end.

Merge sort is like sorting a deck of cards by splitting it in half, sorting each half separately, then merging them back together in order.

Bubble Sort

FeatureDetail
How it worksCompare adjacent pairs, swap if wrong order. Repeat until no swaps needed.
Best caseAlready sorted — 1 pass, N-1 comparisons
Worst caseReverse sorted — N×N comparisons
Simple?Yes — easy to understand and code
Efficient?No — very slow for large lists
python — bubble_sort.py
def bubble_sort(data):
    n = len(data)
    swaps_total = 0
    for pass_num in range(n - 1):
        swapped = False
        for i in range(n - 1 - pass_num):
            if data[i] > data[i + 1]:
                data[i], data[i + 1] = data[i + 1], data[i]  # swap
                swapped = True
                swaps_total += 1
        print(f"  Pass {pass_num + 1}: {data}")
        if not swapped:
            print("  No swaps needed — done early!")
            break
    return swaps_total

numbers = [64, 34, 25, 12, 22, 11, 90]
print(f"Before: {numbers}")
total = bubble_sort(numbers)
print(f"After:  {numbers}")
print(f"Total swaps: {total}")

Merge Sort

FeatureDetail
How it worksSplit list in half repeatedly until single items, then merge them back in order
Best & worst caseAlways N × log₂(N) — consistent performance
Simple?More complex — uses recursion (a function calling itself)
Efficient?Yes — much faster than bubble sort for large lists

python — merge_sort.py
def merge_sort(data):
    if len(data) <= 1:
        return data

    # Split in half
    mid = len(data) // 2
    left = merge_sort(data[:mid])
    right = merge_sort(data[mid:])

    # Merge the two sorted halves
    merged = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            merged.append(left[i])
            i += 1
        else:
            merged.append(right[j])
            j += 1
    merged.extend(left[i:])
    merged.extend(right[j:])
    return merged

numbers = [64, 34, 25, 12, 22, 11, 90]
print(f"Before: {numbers}")
sorted_nums = merge_sort(numbers)
print(f"After:  {sorted_nums}")
